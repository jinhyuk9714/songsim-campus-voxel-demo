이 스타터를 **가톨릭대학교 성심교정**을 모티브로 한 브라우저 기반 3D voxel 캠퍼스 데모로 더 완성도 있게 발전시켜줘.

이 저장소 안에 있는 멀티 에이전트 설정을 그대로 활용해.

## 먼저 읽을 것

- `AGENTS.md`
- `docs/codex/CAMPUS_NOTES.md`
- `src/data/campus.ts`
- 현재 world / interaction / ui 파일 경계

## 멀티 에이전트 작업 방식

먼저 읽기 전용 에이전트를 병렬로 돌리고, 그 다음에 쓰기 에이전트는 한 번에 하나만 사용해.

1. `explorer` 실행
   - 현재 아키텍처 요약
   - 렌더 루프, scene 구성, 파일 소유 경계 정리

2. `layout_planner` 실행
   - 현재 배치가 성심교정처럼 읽히는지 검토
   - 코드 수정 없이 구체적인 개선안 제시

3. `reviewer` 실행
   - 현재 스타터의 런타임 / 인터랙션 / 폴리시 측면 주요 리스크 정리

4. 위 3개 결과를 받은 뒤:
   - 핵심 내용을 합쳐 짧은 실행 계획 제시
   - 쓰기 작업은 한 번에 하나의 write 에이전트만 사용해 진행

## write 에이전트 규칙

- `world_builder`: terrain, roads, buildings, labels, props
- `interaction_builder`: camera, picking, HUD, input
- 두 write 에이전트가 같은 파일을 동시에 수정하면 안 됨
- 수정은 작고 검증 가능하게 나눠서 진행

## 목표 결과물

결과물은 아래처럼 보여야 해:
- 성심교정이라고 알아볼 수 있는 배치
- 포트폴리오에 올릴 만한 품질
- 단순 정적 렌더가 아니라 직접 움직여볼 수 있는 데모
- 앞으로 계속 확장하기 쉬운 구조

## 우선순위

특별한 이유가 없다면 아래 순서로 개선해.

1. 랜드마크 실루엣과 가독성 개선
2. 길 / 경사 / 동선 가독성 개선
3. 카메라 감각과 reset 품질 개선
4. 건물 선택 피드백과 정보 패널 개선
5. 장식 요소는 과하지 않게 보강
6. 마지막에 임팩트 있는 기능 하나 추가
   - 랜드마크 fly-to 카메라
   - 낮/밤 전환
   - 미니맵
   - 가이드 투어 루트

## 마무리

- build 실행
- test 실행
- 파일별 변경 내용 요약
- 남은 아쉬운 점 / 다음 추천 작업 정리
